# 图标自适应屏幕尺寸更新说明

## 📅 修改日期
2026 年 3 月 17 日

## 🎯 修改目标
解决图标在不同屏幕尺寸设备上显示大小不一致的问题，实现图标尺寸自适应。

---

## 📝 修改内容

### 1. **Icons.ets 组件升级**

**文件路径**: `entry/src/main/ets/common/Icons.ets`

#### 新增功能
- ✅ 新增 `IconSizeType` 类型定义：`'nav' | 'list' | 'card' | 'empty' | 'header' | 'custom'`
- ✅ 新增 `sizeType` 属性，默认值为 `'custom'`（兼容旧代码）
- ✅ 新增 `getScreenWidth()` 方法：获取设备屏幕宽度
- ✅ 新增 `getAdaptiveSize()` 方法：根据屏幕宽度计算自适应尺寸

#### 基准尺寸配置（以 360px 屏幕为基准）
| 类型 | 基准尺寸 | 适配范围 | 使用场景 |
|------|---------|---------|---------|
| `nav` | 32px | 27-37px | 底部导航图标 |
| `list` | 26px | 22-30px | 列表项图标 |
| `card` | 32px | 27-37px | 卡片状态图标 |
| `empty` | 56px | 48-64px | 空状态大图标 |
| `header` | 36px | 31-41px | 页面标题旁图标 |
| `custom` | 24px | 固定值 | 自定义尺寸（兼容旧代码） |

#### 适配算法
```typescript
// 屏幕缩放系数 = 当前屏幕宽度 / 基准屏幕宽度 (360px)
const scaleFactor = screenWidth / 360;

// 限制缩放范围在 ±15% 内
const clampedFactor = Math.max(0.85, Math.min(1.15, scaleFactor));

// 最终尺寸 = 基准尺寸 × 缩放系数
return Math.round(baseSize * clampedFactor);
```

## 🔧 兼容性说明

### 向后兼容
- ✅ 保留 `iconSize` 属性，默认值 24px
- ✅ 默认 `sizeType='custom'`，使用 `iconSize` 值
- ✅ 旧代码无需修改仍可正常运行

### 回退方案
如需回退到旧版固定像素模式，有两种方式：

**方式 1**: 修改单个图标调用
```typescript
// 改回使用 iconSize
SvgIcon({ name: 'home', color: '#2563eb', iconSize: 28 })
```

**方式 2**: 修改 Icons.ets 组件
```typescript
// 在 getAdaptiveSize() 方法中，始终返回 iconSize
private getAdaptiveSize(): number {
  return this.iconSize;  // 强制使用固定像素
}
```

## 📊 预期效果

### 不同设备上的图标尺寸

| 设备类型 | 屏幕宽度 | nav 图标 | list 图标 | card 图标 | empty 图标 |
|---------|---------|---------|----------|----------|-----------|
| 小屏手机 | 320px | 27px | 22px | 27px | 48px |
| 标准手机 | 360px | 32px | 26px | 32px | 56px |
| 大屏手机 | 412px | 37px | 30px | 37px | 64px |
| 平板 | 720px | 37px* | 30px* | 37px* | 64px* |

*注：平板设备因缩放系数限制在±15%，实际使用最大允许值

## ✅ 测试建议

1. **多设备测试**
   - 在小屏手机（320px 宽度）上测试图标是否清晰可见
   - 在大屏手机（412px+ 宽度）上测试图标是否过大
   - 在平板设备上测试整体布局协调性

2. **页面测试**
   - 主页：检查视频区域、状态卡片图标大小
   - 个人页：检查头像、菜单列表图标大小
   - 记录页：检查空状态、列表项图标大小
   - 底部导航：检查三个 Tab 图标大小

3. **方向测试**
   - 横屏/竖屏切换时图标尺寸是否正确更新

---

## 📌 注意事项

1. **性能影响**: 每次渲染都会调用 `getScreenWidth()`，但该方法有 try-catch 保护，性能影响可忽略

2. **降级处理**: 如果获取屏幕宽度失败，会使用基准值 360px，确保图标正常显示

3. **后续优化**: 可将屏幕宽度缓存为 `@State`，在屏幕方向变化时更新

2. 图标尺寸是否符合预期（参考上方尺寸表）
3. 必要时可使用回退方案临时恢复
