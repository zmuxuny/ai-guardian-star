## 🎯 使用方法

### 用户操作
1. 打开应用
2. 切换到「个人」Tab
3. 点击「深色模式」开关
4. 整个应用立即切换主题

### 开发者集成
```typescript
// 1. 导入主题管理器
import { ThemeManager } from '../common/ThemeManager';

// 2. 在组件中获取实例
@State themeManager: ThemeManager = ThemeManager.getInstance();

// 3. 使用主题颜色
.fontColor(this.themeManager.colors.primaryTextColor)
.backgroundColor(this.themeManager.colors.backgroundColor)
```

## 🎨 设计原则

### 1. 颜色对比度
- 确保文字与背景有足够对比度
- 深色模式使用柔和的白色（#EBEBF5）而非纯白
- 浅色模式使用接近黑的深色（#1C1C1E）而非纯黑

### 2. 语义化颜色
```typescript
primaryTextColor      // 主要内容文字
secondaryTextColor    // 次要说明文字
tertiaryTextColor     // 辅助提示文字
primaryButtonColor    // 主操作按钮
dangerButtonColor     // 危险操作按钮
```

### 3. 一致性
- 所有页面使用相同的颜色变量
- 保持品牌色不变（如红色警告）
- 图标颜色随主题自适应

## 📊 技术特点

### 响应式架构
```typescript
@Observed              // 类级别可观察
export class ThemeManager {
  @Track public isDarkMode: boolean;     // 自动追踪变化
  @Track public colors: ThemeColors;     // 自动追踪变化
}
```

### 本地持久化
```typescript
// 保存主题偏好
await dataPreferences.put('isDarkMode', this.isDarkMode);
await dataPreferences.flush();

// 加载主题偏好
const savedTheme = await dataPreferences.get('isDarkMode', false);
```

### 系统集成
```typescript
// 更新状态栏颜色
await mainWindow.setWindowSystemBarProperties({
  statusBarContentColor: this.isDarkMode ? '#FFFFFF' : '#000000'
});
```

## 🔄 工作流程

```
用户切换开关
    ↓
ThemeManager.setTheme(isDark)
    ↓
更新 isDarkMode 状态
    ↓
应用新的主题颜色
    ↓
保存到本地存储
    ↓
更新系统状态栏
    ↓
所有页面自动重新渲染
```

## 📈 性能优化

1. **单例模式**: 全局唯一实例，减少内存开销
2. **响应式更新**: 只更新变化的部分，避免全量刷新
3. **延迟初始化**: 应用启动时异步加载主题偏好
4. **缓存颜色对象**: 避免重复创建颜色实例
