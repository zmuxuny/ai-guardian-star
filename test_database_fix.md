# 数据库修复验证步骤

## 🔧 已修复的问题

### 1. 用户名传递不一致问题
- **问题**：写入数据库时使用 `'default_user'`，查询时使用空字符串 `''`
- **修复**：创建 `UserManager` 统一管理用户名获取
- **影响文件**：
  - `MqttManager.ets` - 写入健康事件时
  - `person.ets` - 查询用户统计时  
  - `HealthHistory.ets` - 查询历史记录时
  - `Login.ets` - 登录成功后设置用户名
  - `DatabaseDiagnostic.ets` - 诊断工具显示当前用户

## 🧪 验证步骤

### 步骤 1：清除旧数据（可选）
1. 打开 App
2. 在登录页**长按 Logo** 进入数据库诊断工具
3. 点击"清除所有数据"
4. 确认清除

### 步骤 2：重新登录测试
1. 返回登录页
2. 输入账号密码登录
3. 观察控制台日志输出

### 步骤 3：连接板子测试数据保存
1. 确保 MQTT 连接正常
2. 模拟摔倒/久坐事件
3. 观察控制台日志：
   ```
   [mqtt]:✅ 摔倒记录已添加 2026.03.27 10:30:45 (user=张三)
   [mqtt]:📦 ArkDB 写入摔倒事件成功 rowId=1 (user=张三)
   ```

### 步骤 4：验证数据查询
1. 打开"个人"页
2. 查看"累计预警次数"是否更新
3. 点击"历史记录"
4. 查看是否有摔倒/久坐记录

### 步骤 5：使用诊断工具验证
1. 返回登录页
2. **长按 Logo** 进入数据库诊断工具
3. 查看：
   - 当前登录用户显示正确
   - 总事件数 > 0
   - 当前用户事件数 > 0
   - 日志显示查询成功

## 📊 预期结果

### 修复前
- 登录记忆功能：✅ 正常
- 健康信息保存：❌ 失败
- 数据库数据：❌ 看不到

### 修复后  
- 登录记忆功能：✅ 正常
- 健康信息保存：✅ 正常
- 数据库数据：✅ 可以看到

## 🔍 关键日志关键字

| 日志前缀 | 含义 |
|---------|------|
| `[mqtt]:✅` | MQTT 记录添加成功（内存） |
| `[mqtt]:📦` | ArkDB 写入成功 |
| `[mqtt]:❌` | ArkDB 写入失败 |
| `[person]:` | 个人页面数据加载 |
| `[HealthHistory]:` | 历史记录页数据加载 |
| `DatabaseHelper` | 数据库核心操作 |

## 🚨 如果仍有问题

1. **检查数据库初始化**
   - 查看 `EntryAbility.ets` 中的初始化日志
   - 确认没有 "DB init failed" 错误

2. **检查用户登录态**
   - 在诊断工具中查看"当前登录用户"
   - 确认不是 `default_user` 或空

3. **检查 MQTT 连接**
   - 确认板子连接正常
   - 查看 `[mqtt]:` 开头的日志

4. **手动触发测试**
   - 可以在诊断工具中执行"刷新"
   - 观察是否有数据查询失败
