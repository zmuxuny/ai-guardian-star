# 数据库修复说明

**最后更新：** 2026-03-27  
**SDK 版本：** API 20 (HarmonyOS 6.0.0)

## 问题描述

登录后连接板子，摔倒/久坐记录未保存到数据库，IDE 中看不到数据库文件。

### 根因分析

1. **用户名未正确传递**：`MqttManager.ets` 写入数据时用户名可能为空字符串，导致数据归属混乱
2. **查询逻辑不一致**：写入时 `username = ''`，查询时传入非空 username，查不到数据
3. **缺少调试日志**：数据库操作没有详细日志，无法追踪数据流向

## 已修复内容

### 1. MqttManager.ets — 统一用户名获取

```typescript
private getCurrentUsername(): string {
  return AppStorage.get<string>('loggedInUsername') ?? 'default_user';
}
```
默认值从 `''` 改为 `'default_user'`，添加详细日志输出。

### 2. DatabaseHelper.ets — 查询逻辑优化

- 插入时确保 username 不为空
- 查询时若 username 为空则查所有数据（兼容旧数据）
- 新增 `getRdbStore()` 公开方法供诊断工具使用
- 新增 `queryAllHealthEventsForDebug()` 诊断方法

```typescript
if (username && username.trim() !== '') {
  predicates.equalTo('username', username);
} else {
  // 不设 username 过滤，查所有数据（兼容旧数据）
}
```

### 3. person.ets — 数据加载修复

- `pathStack` 从 `NavPathStack | undefined` 改为 `NavPathStack = new NavPathStack()`
- 从数据库加载用户信息和累计预警次数
- 添加详细日志和错误处理

### 4. HealthHistory.ets — SDK 20 规范修正

- `@Builder` 函数添加 `(): void` 返回类型
- `LoadingProgress` 用 `Column` 包裹
- 所有回调使用箭头函数

### 5. DatabaseDiagnostic.ets — 新建诊断工具

**入口**：登录页长按 Logo

**功能**：
- 查看所有健康事件 / 当前用户事件
- 查看所有用户列表
- 清除所有数据（危险操作）

### 6. SDK 20 代码规范要点

```typescript
// ✅ 使用公开方法而非私有方法
const store = db.getRdbStore()  // 而非 db['getStore']()

// ✅ 显式检查关闭，不用可选链
if (resultSet !== undefined) { resultSet.close() }

// ✅ @Builder 必须声明返回类型
@Builder
export function HealthHistoryBuilder(): void { ... }

// ✅ pathStack 必须初始化
pathStack: NavPathStack = new NavPathStack();
```

## 验证步骤

### 1. 清除旧数据（可选）
登录页长按 Logo → 进入诊断工具 → 清除所有数据

### 2. 登录测试
输入账号密码登录，观察控制台日志

### 3. 连接板子测试
确保 MQTT 连接正常，模拟摔倒/久坐事件，观察日志：
```
[mqtt]:✅ 摔倒记录已添加 2026.03.27 10:30:45 (user=张三)
[mqtt]:📦 ArkDB 写入摔倒事件成功 rowId=1 (user=张三)
```

### 4. 验证数据查询
- 个人页 → 查看"累计预警次数"是否更新
- 历史记录 → 查看记录列表是否显示

### 5. 诊断工具验证
- 长按 Logo 进入诊断工具
- 确认当前登录用户显示正确、事件数 > 0

## 日志关键字

| 日志前缀 | 含义 |
|---------|------|
| `[mqtt]:✅` | MQTT 记录添加成功（内存） |
| `[mqtt]:📦` | ArkDB 写入成功 |
| `[mqtt]:❌` | ArkDB 写入失败 |
| `[person]:` | 个人页面数据加载 |
| `[HealthHistory]:` | 历史记录页数据加载 |
| `DatabaseHelper` | 数据库核心操作 |

## 常见问题排查

### 数据库文件在哪？
设备路径：`/data/storage/el2/base/dependencies_cache/GuardianStar.db`  
查看方式：DevEco Studio → Device File Explorer → 导航到该路径

### 登录后看不到数据？
进入诊断工具 → 查看"当前登录用户"是否正确 → 查看"所有健康事件"是否有数据

### 连接板子后没有记录？
检查控制台 `[mqtt]:` 日志 → 确认 MQTT 连接状态 → 确认是否收到消息

### 编译报错找不到 relationalStore？
检查 `build-profile.json5` 中 `targetSdkVersion` 为 `6.0.0(20)`
