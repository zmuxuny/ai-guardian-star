# 数据库加载问题修复说明

**修复日期：** 2026-03-24  
**问题描述：** 登录后连接板子，摔倒/久坐记录未保存到数据库，IDE 中看不到数据库文件

---

## 🔍 问题分析

### 核心问题

1. **用户名未正确传递到数据库**
   - `MqttManager.ets` 中写入数据时使用 `AppStorage.get('loggedInUsername') ?? ''`
   - 如果用户未登录或登录态未同步，`username` 会是空字符串
   - 空字符串作为默认值会导致数据归属混乱

2. **查询逻辑不一致**
   - 写入时 `username = ''`（空字符串）
   - 查询时传入非空 username，导致查不到数据

3. **缺少调试日志**
   - 数据库操作没有详细日志，无法追踪数据流向

### 3. `person.ets` 修改

**文件路径：** `entry/src/main/ets/pages/person.ets`

```typescript
aboutToAppear(): void {
  const currentUser = AppStorage.get<string>('loggedInUsername') ?? '';
  const db = DatabaseHelper.getInstance();
  
  // 从数据库加载当前用户信息
  db.queryUserByUsername(currentUser).then((user: UserInfo | null) => {
    if (user) {
      this.userName = user.username;
      const msPerDay = 24 * 60 * 60 * 1000;
      this.useDays = Math.max(1, Math.floor((Date.now() - user.createTime) / msPerDay));
      console.log(`[person]: 用户信息加载成功 user=${user.username}, useDays=${this.useDays}`);
    } else {
      console.warn(`[person]: 未找到用户信息 user=${currentUser}`);
    }
  }).catch((err) => {
    console.error(`[person]: 查询用户失败 ${err}`);
  });
  
  // 从数据库加载当前用户的累计预警次数
  db.queryHealthEvents('fall', currentUser)
    .then((events: HealthEvent[]) => { 
      this.totalAlertCount = events.length;
      console.log(`[person]: 摔倒事件加载成功 count=${events.length}, user=${currentUser}`);
    })
    .catch((err) => { 
      this.totalAlertCount = this.mqttManager.fallRecords.length;
      console.error(`[person]: 查询健康事件失败 ${err}`);
    });
}
```

**修改点：**
- 添加详细的日志输出
- 错误处理更明确

### 4. `HealthHistory.ets` 修改

**文件路径：** `entry/src/main/ets/pages/HealthHistory.ets`

```typescript
private loadHistory(): void {
  this.isLoading = true;
  const db = DatabaseHelper.getInstance();
  const currentUser = AppStorage.get<string>('loggedInUsername') ?? '';
  
  console.log(`[HealthHistory]: 开始加载数据 user=${currentUser}`);

  db.queryHealthEvents('fall', currentUser).then((events: HealthEvent[]) => {
    this.fallRecords = events.map((e: HealthEvent): FallRecord => ({
      time: e.timeStr,
      date: e.dateStr,
      timestamp: e.eventTime,
    }));
    console.log(`[HealthHistory]: 摔倒记录加载完成 count=${events.length}`);
  }).catch((err) => {
    this.fallRecords = [];

  });
```

## 🔧 常见问题排查

### Q1: 数据库文件在哪里？

**路径：**
```
/data/storage/el2/base/dependencies_cache/GuardianStar.db
```

**查看方法：**
1. DevEco Studio → Device File Explorer
2. 连接设备
3. 导航到上述路径

---

### Q2: 登录后看不到数据？

**检查步骤：**
1. 打开诊断工具，查看"当前登录用户"
2. 确认用户名是否正确
3. 查看"所有健康事件"中是否有数据
4. 如果有数据但 username 不对，说明登录态同步有问题

---

### Q3: 连接板子后没有记录？

**检查步骤：**
1. 查看控制台 `[mqtt]:` 开头的日志
2. 确认 MQTT 是否连接成功
3. 确认是否收到消息
4. 检查 `addFallRecord()` 是否被调用

---

### Q4: 诊断工具打不开？

**解决方法：**
1. 确保已导入 `DatabaseDiagnostic.ets` 到项目
2. 在 `Login.ets` 中检查 `pathStack` 是否正确
3. 查看控制台是否有路由错误
