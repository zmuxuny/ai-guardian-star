# 贡献指南 (CONTRIBUTING)

感谢你关注并参与本项目！为保持仓库质量与协作高效，请在贡献前阅读以下流程与规范：

## 贡献流程
1. 先在 Issues 中提交问题或功能请求，便于讨论与规划。
2. Fork 本仓库并在本地创建分支，分支命名建议使用 `feat/xxx`、`fix/xxx` 或 `docs/xxx`。
3. 在分支完成开发后，发起 Pull Request（PR），并在 PR 描述中写明变更目的、关联 Issue、测试结果等。

## 提交规范
- 使用清晰的提交信息，推荐使用 Conventional Commits 风格，例如： `feat: 添加新的MQTT客户端`、`fix: 修复崩溃问题`。
- 每个 PR 应包含合理的测试或复现步骤。

## 代码风格
- 请遵循项目中已有的代码风格与 lint 规则。
- 若新增重要模块，请补充对应的单元测试与文档说明。

## 其他
- 若变更涉及许可、第三方依赖或重大架构修改，请先通过 Issue 与维护者确认。
- 如需添加大文件（>50MB），请联系维护者使用 LFS 或其他方案。

感谢你的贡献！欢迎在 PR 中 @ 指定维护者进行 Review 请求。
