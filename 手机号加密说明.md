## 🔒 加密规则

**显示格式：** 保留前 3 位和后 4 位，中间用 `*` 替代

**示例：**
- `13812345678` → `138****5678`
- `1391234567` → `139**567`
- `137123456789` → `137*****5678`

### 2. 新增加密方法

```typescript
/** 手机号加密处理：只显示前 3 位和后 4 位，中间用 * 替代 */
private encryptPhone(phone: string): string {
  if (!phone || phone.length < 7) {
    return phone;
  }
  // 保留前 3 位和后 4 位，中间用 * 替代
  const prefix = phone.substring(0, 3);
  const suffix = phone.substring(phone.length - 4);
  const maskLength = phone.length - 7;
  const mask = '*'.repeat(maskLength);
  return prefix + mask + suffix;
}
```

### 3. 加载用户信息时加密

```typescript
private loadUserInfo(): void {
  const currentUser = AppStorage.get<string>('loggedInUsername') ?? '';
  const db = DatabaseHelper.getInstance();

  db.queryUserByUsername(currentUser)
    .then((user: UserInfo | null) => {
      if (user) {
        this.userName = user.username;
        // 加密手机号
        if (user.phone) {
          this.userPhone = this.encryptPhone(user.phone);
        }
        // ...
      }
    });
}
```

## 🔐 隐私保护说明

1. **仅前端加密显示** - 数据库中仍存储完整手机号
2. **不影响功能** - 所有需要手机号的逻辑（如登录、验证）不受影响
3. **符合规范** - 符合个人信息保护相关要求

---

## 📝 注意事项

1. **手机号长度校验** - 小于 7 位的手机号不加密（兼容特殊情况）
2. **空值处理** - 如果用户没有手机号，显示默认的 `userId`
3. **图标前缀** - 加密手机号前添加 📱 图标，提升视觉识别度
